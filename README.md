# TabEx
多标签页文件浏览器，类似 Clover 的 Windows 资源管理器增强工具

## 📥 下载

**最新版本**: 从 [Releases](../../releases/latest) 页面下载 `TabExplorer.exe`

下载即用，无需安装 Python 环境！

---

## ✨ 核心功能

- ✅ **多标签页浏览** - Chrome 风格标签页，独立历史记录，支持拖拽排序
- ✅ **AI 助手侧边栏** - 可选启用的右侧聊天面板，支持目录操作、文件读写、脚本执行和安全确认
- ✅ **高 DPI 自适应** - 自动检测屏幕 DPI 并动态缩放 UI，高分屏显示清晰
- ✅ **标签页固定** - 固定常用文件夹，开机自动恢复
- ✅ **TortoiseGit 集成** - 标题栏快捷按钮，一键打开 Log/Commit/Git Bash（需安装 TortoiseGit 或 Git for Windows）
- ✅ **标题栏启动区** - 支持拖入应用、快捷方式或脚本（`.lnk/.exe/.bat/.cmd/.ps1`），一键启动、右键删除、拖拽排序（最多 20 个），并尽量解析目标程序图标避免快捷方式箭头覆盖
- ✅ **标题栏工具按钮** - 一键在当前标签页路径打开 cmd、PowerShell、Git Bash，或快速打开计算器
- ✅ **会话恢复** - 关闭时自动缓存标签，启动时智能恢复工作环境，支持窗口恢复提示
- ✅ **当前目录快捷定位** - `Ctrl+G` 按关键字检索当前目录下的文件夹和文件名，多结果使用表格选择，并在当前目录直接选中目标
- ✅ **快捷导航** - 前进/后退、面包屑导航、智能路径补全、路径栏智能压缩
- ✅ **Everything 搜索** - 毫秒级全盘搜索，速度提升 100-1000 倍
- ✅ **书签管理** - 层级书签、导入/导出
- ✅ **轻量级提示** - 右下角 Toast 消息，5秒自动消失，无需点击确认
- ✅ **自动监听 Explorer** - 新打开的资源管理器窗口自动嵌入为标签页
- ✅ **误操作保护** - Ctrl+Shift+T 恢复关闭的标签页（最多 20 个）
- ✅ **文件系统监控** - 文件变化自动刷新（watcher + 低频轮询兜底），支持回收站恢复
- ✅ **智能自动刷新** - 当前标签主动监听文件修改并刷新，后台标签自动降频，兼顾及时性与性能
- ✅ **智能状态栏** - 显示目录项目数、选中文件信息、总大小、修改时间等
- ✅ **全面拖拽支持** - 文件夹拖拽到标签栏、目录树任意位置打开
- ✅ **单实例模式** - 右键菜单打开文件夹时在已有窗口新建标签页
- ✅ **性能优化** - 大文件夹异步加载、分割器拖动节流、响应式布局
- ✅ **长期运行稳定性优化** - 搜索窗口、Explorer 监控、标签页资源、聊天线程与定时器均增加清理与回收路径，降低长时间运行后的内存和句柄堆积风险
- ✅ **资源快照日志** - 可在设置中启用定时资源快照，持续记录内存、线程数、标签页数等运行指标，便于长稳排查

## 🆕 最近更新

### v3.27 (2026-05-15)
- **标题栏工具按钮图标改为文字标签**：
  - CMD / PS / CAL / GB 四个快捷按钮从原 emoji 图标（🪟 ⚡ 🧮 🐚）改为文字标签，避免字体差异导致图标显示异常
  - 四个按钮各自使用不同颜色（CMD 深黑 / PS 亮蓝 / CAL 橙红 / GB 橙色），视觉区分更直观
  - 按钮字体自动缩小以适配固定按钮尺寸，确保三字符文本完整显示
- **修复 AI 助手面板"当前目录"不随目录切换更新的问题**：
  - 根本原因：原实现只在"切换标签页"或"打开 AI 面板"时刷新当前目录提示，未覆盖"在同一标签页内进行目录导航"的场景
  - 修复：在 `_perform_navigation`（地址栏输入/面包屑点击导航）、`_on_shell_navigate_complete`（双击文件夹 NavigateComplete2 信号），以及备用 MetaCall 通道三处均补充当前目录同步，确保任何导航方式都能即时刷新 AI 面板顶部的"当前目录"提示和 AI 上下文

### v3.26 (2026-05-12)
- **优化双击空白处返回上一级的响应速度**：
  - 问题：原实现叠加两层延迟定时器（400 ms + 700 ms），最长需等待 1100 ms 才触发返回
  - 修复：在 `MouseButtonDblClick` 事件发生时立即对双击坐标执行 `LVM_HITTEST` native 命中测试，点中文件/文件夹项目时 Explorer 自行处理，点中空白区域时仅等待 50 ms 后执行 go_up
  - 效果：空白区双击返回几乎即时，与双击文件夹进入、双击文件打开完全不冲突

### v3.25 (2026-05-12)
- **修复打开映射网络驱动器（mapped network drive）时软件卡死的问题**：
  - 根本原因：`_is_slow_path()` 仅检测 UNC 路径和 OneDrive，未识别通过盘符映射的网络驱动器；网络抖动时 `os.scandir()` 在主线程上无限期阻塞
  - 修复：使用 `GetDriveTypeW` API 检测 `DRIVE_REMOTE`（类型 4），映射驱动器统一视为慢速路径，所有 `os.scandir()` 保护点自动生效
  - 同时修复 `_count_dir_entries()`（状态栏目录计数）缺少慢速路径检查的问题

### v3.24 (2026-05-12)
- **性能优化：对话历史写入防抖**：`ChatPanel` 每次追加气泡时改为 500 ms 防抖写入，避免连续追加时反复触发 JSON 序列化与磁盘 I/O
- **性能优化：QTextBrowser 显示块数量上限**：聊天显示框累计 block 超过 400 时自动重建，防止长时间使用后渲染耗时持续增长
- **性能优化：定时器资源释放**：`FileExplorerTab.cleanup()` 停止各定时器后补充调用 `deleteLater()` 并置 `None`，确保 Qt 对象树彻底释放
- **性能优化：`is_key_pressed` 闭包消除**：将 `ctypes.windll.user32.GetAsyncKeyState` 提取为局部变量 `_GetAsyncKeyState`，避免每次 160 ms 轮询时重建闭包与属性查找链
- **性能优化：后台标签页路径同步跳过**：`sync_path_bar_with_explorer` 检测到本标签非当前活跃标签时，立即停止同步定时器并返回，消除后台标签的持续 COM 跨进程调用

### v3.23 (2026-05-12)
- **修复搜索对话框关闭后约 43 MB 内存未释放的问题**：
  - 根本原因：`SearchDialog` 未设置 `Qt.WA_DeleteOnClose`，导致关闭后 Qt C++ 对象（含所有子控件）不会自动销毁，持续占用内存
  - 修复：在 `SearchDialog.__init__` 加入 `self.setAttribute(Qt.WA_DeleteOnClose)`，关闭即销毁整个控件树，内存即时归还

## 🛠️ 运行环境

**从可执行文件运行**（推荐）：
1. 从 [Releases](../../releases/latest) 下载 `TabExplorer.exe`
2. 双击运行即可
3. 程序会在 `TabExplorer.exe` 同目录读写以下持久化文件：
  - `config.json`：应用设置、固定标签、标题栏快捷方式、会话缓存
  - `bookmarks.json`：书签数据
  - `chat_history.json`：AI 聊天历史
  - `runtime_health.log`：资源快照日志（仅在设置中启用后生成）

**从源码运行**（需要 Python 3.9+）：
1. 双击 `0_install_requirements.bat` 安装依赖
2. 双击 `1_TabEx.bat` 启动程序
3. 若使用 AI 功能，确保 `requests` 已安装并在 `config.json` 或设置界面中填写 AI 服务配置

**开机自启动**：
将 `1_TabEx.bat` 的快捷方式放到 `shell:startup` 目录

## 📖 主要功能说明

### Everything 搜索（推荐）
1. 下载安装 [Everything](https://www.voidtools.com/)
2. 下载 [es.exe](https://www.voidtools.com/downloads/) 并放到 Everything 安装目录
3. 启动 Everything 并等待索引完成
4. 在 TabExplorer 中按 `Ctrl+F` 打开搜索，勾选"使用 Everything"即可享受毫秒级搜索

### 自动监听 Explorer 窗口
- 类似 Clover 的核心功能
- 新打开的 Windows 资源管理器窗口自动嵌入为标签页
- 可在设置中启用/禁用
- 若嵌入式 `Shell.Explorer` 控件初始化失败，会自动回退为使用系统资源管理器打开，避免生成异常标签页或拖慢界面

### 智能自动刷新
- 当前标签页会同时使用目录 watcher、文件 watcher 和低频轮询兜底检测文件变更
- 编辑当前目录里的文件后，列表、修改日期、大小等信息会更可靠地自动刷新
- 后台标签页不会持续高频轮询，只在切回时按需补刷新
- 这种策略兼顾了及时刷新和多标签场景下的性能开销
- 当当前目录正好是程序运行目录时，会自动忽略应用自身写出的配置/日志文件变化，避免 watcher 与会话快照互相触发导致刷新风暴

### 当前目录快捷定位
- 按 `Ctrl+G` 可输入关键字，快速检索当前目录下名称中包含关键字的文件夹和文件
- 只有一个匹配结果时会直接在当前目录中选中该项目
- 有多个匹配结果时会弹出表格选择窗口，展示名称、类型和完整路径
- 该快捷键可在设置界面中单独启用或禁用

### AI 助手
- 在 `设置 → AI 助手设置` 中配置启用开关、API 地址、API Key、模型名、面板宽度和系统提示词
- 标题栏 `🤖` 按钮仅在 AI 开关启用时显示
- 支持 OpenAI 兼容接口，也支持本地/内网模型服务
- AI 默认只在用户明确要求时输出操作指令，避免普通问答误操作

### 资源快照日志
- 在 `设置 → 调试设置` 中可启用“资源快照日志”
- 可设置采样间隔（分钟），默认用于低频长期观测，不建议设置过短
- 启用后程序会定时写入 `runtime_health.log`
- 可直接在设置中点击“打开资源日志”查看当前日志文件或日志目录
- 日志内容包括内存占用、线程数、标签页数量、搜索窗口数量、Toast 数量和 AI 工作线程状态

**当前支持的 AI 指令：**
- `OPEN_DIR`：打开或切换目录
- `READ_FILE`：读取文件内容，自动分段读取大文件
- `LIST_DIR`：列出目录内容
- `MKDIR`：创建目录
- `WRITE_FILE`：写入/覆盖文件
- `DELETE`：删除文件或目录（需确认）
- `RUN_SCRIPT`：运行脚本（需确认）

**安全限制：**
- AI 文件操作仅允许在当前标签页目录范围内执行
- 覆盖文件、删除文件/目录、运行脚本都需要用户手动确认
- 聊天面板支持拖拽本地文件进行快速阅读，单文件读取大小有限制

### 会话恢复（新）
- 关闭软件时自动保存所有非固定标签页
- 下次启动时自动恢复，继续上次的工作
- 恢复顺序：先加载固定标签，再恢复缓存标签
- 可在设置中启用/禁用（默认开启）
- 打包版与源码版都会将会话缓存持久化到各自运行目录下的 `config.json`

### 书签管理
- 支持文件夹层级组织
- 导入/导出 JSON 格式书签
- 兼容 Chrome 书签格式
  

### 标签页历史
- 最多记录 20 个关闭的标签页
- `Ctrl+Shift+T` 快速恢复
- 或点击标签栏右侧的 ↶ 按钮

### 文件复制快捷键
- **Alt+Z** - 快速复制选中文件名（如果选择多个文件，会用逗号分隔）
- **Alt+X** - 快速复制文件路径\文件名（路径分隔符可在设置中配置为 `/` 或 `\`）
- 两个快捷键都支持多文件批量拷贝
- `Alt+Z`、`Alt+X`、`Ctrl+G` 等快捷键都可在设置界面中单独启用/禁用

## 🎨 图标维护

### 图标文件
- `TabExplorer.ico` - 应用图标，支持多分辨率（256, 128, 96, 64, 48, 32, 24, 18, 16 px）
- `generate_icon.py` - 图标生成脚本，基于 Pillow

### 修改图标设计
1. 编辑 `generate_icon.py` 中的 `create_te_icon()` 函数，修改色彩、边框、文字等参数
2. 运行脚本生成新图标：
   ```bash
   python generate_icon.py
   ```
3. 图标将自动保存为 `TabExplorer.ico`
4. 如需打包 exe，运行 `2_build_exe.bat` 即可

### 图标设计原则
- **所有尺寸统一比例**：基于 18px 标准，边框占 11%，字体大小占 65%
- **外形**：方形，无圆角（仅内部白框有 1px 圆角）
- **配色**：蓝色背景（#2196F3）+ 白色内框 + 蓝色 TE 文字
- **多尺寸**：使用 LANCZOS 重采样确保缩小后清晰
