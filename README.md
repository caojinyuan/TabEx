# TabEx
多标签页文件浏览器，类似 Clover 的 Windows 资源管理器增强工具

## 📥 下载

**最新版本**: 从 [Releases](../../releases/latest) 页面下载 `TabExplorer.exe`

下载即用，无需安装 Python 环境！

---

## ✨ 核心功能

- ✅ **多标签页浏览** - Chrome 风格标签页，独立历史记录，支持拖拽排序
- ✅ **左右分屏对比** - 按 `F3` 或标题栏 `◫` 按钮，把当前标签移到右侧独立标签组（右侧标签栏与左侧并排于书签栏上方，支持双击新建、关闭、拖拽等原生标签功能）；再次 `F3` 把右侧标签移回左侧并关闭分屏，便于两个文件夹并排对比与互拖文件
- ✅ **鼠标手势** - 文件区按住右键画线快速导航（后退/前进/新建/关闭标签等），带实时轨迹与方向提示，可在设置中开关
- ✅ **AI 助手侧边栏** - 可选启用的右侧聊天面板，支持目录操作、文件读写、脚本执行和安全确认
- ✅ **高 DPI 自适应** - 自动检测屏幕 DPI 并动态缩放 UI，高分屏显示清晰
- ✅ **标签页固定** - 固定常用文件夹，开机自动恢复
- ✅ **TortoiseGit 集成** - 标题栏快捷按钮，一键打开 Log/Commit/Git Bash（需安装 TortoiseGit 或 Git for Windows），支持变更文件图标修改功能
- ✅ **标题栏启动区** - 支持拖入应用、快捷方式或脚本（`.lnk/.exe/.bat/.cmd/.ps1`），一键启动、右键删除、拖拽排序（最多 20 个），并尽量解析目标程序图标避免快捷方式箭头覆盖
- ✅ **标题栏工具按钮** - 一键在当前标签页路径打开 cmd、PowerShell、Git Bash，或快速打开计算器
- ✅ **会话恢复** - 关闭时自动缓存标签，启动时智能恢复工作环境，支持窗口恢复提示
- ✅ **当前目录快捷定位** - `Ctrl+G` 按关键字检索当前目录下的文件夹和文件名，多结果使用表格选择，并在当前目录直接选中目标
- ✅ **快捷导航** - 前进/后退、面包屑导航（段间 `›` 下拉快速切换同级目录）、编辑模式路径自动补全、路径栏智能压缩
- ✅ **Everything 搜索** - 毫秒级全盘搜索，速度提升 100-1000 倍
- ✅ **书签管理** - 层级书签、导入/导出
- ✅ **轻量级提示** - 右下角 Toast 消息，5秒自动消失，无需点击确认
- ✅ **自动监听 Explorer** - 新打开的资源管理器窗口自动嵌入为标签页
- ✅ **误操作保护** - Ctrl+Shift+T 恢复关闭的标签页（最多 20 个）
- ✅ **文件系统监控** - 文件变化自动刷新（watcher + 低频轮询兜底），支持回收站恢复
- ✅ **智能自动刷新** - 当前标签主动监听文件修改并刷新，后台标签自动降频，兼顾及时性与性能
- ✅ **智能状态栏** - 显示目录项目数、选中文件信息、总大小、修改时间；Git 仓库下实时显示分支名和暂存/修改/未跟踪文件数（颜色标注，后台异步不阻塞 UI）
- ✅ **全面拖拽支持** - 文件夹拖拽到标签栏、目录树任意位置打开
- ✅ **单实例模式** - 右键菜单打开文件夹时在已有窗口新建标签页
- ✅ **性能优化** - 大文件夹异步加载、分割器拖动节流、响应式布局
- ✅ **长期运行稳定性优化** - 搜索窗口、Explorer 监控、标签页资源、聊天线程与定时器均增加清理与回收路径，降低长时间运行后的内存和句柄堆积风险
- ✅ **资源快照日志** - 可在设置中启用定时资源快照，持续记录内存、线程数、标签页数等运行指标，便于长稳排查

## 🆕 最近更新(只保留最新)

### v3.59 (2026-06-30)
- **新增：左右分屏对比**：按 `F3` 或标题栏 `◫` 按钮，把当前标签“移动”到右侧独立标签组（连同其嵌入的资源管理器、历史与状态一起迁移，非复制）；右侧标签栏与左侧并排于书签栏上方，且支持双击空白处新建标签、关闭按钮、拖拽文件夹打开等与左侧一致的原生标签功能；左侧自动切换到其它标签（若原本只有一个标签则自动新建一个）；再次 `F3` 把右侧标签全部移回左侧并关闭分屏
- **分屏面板为一等浏览面板**：左右两侧都完整支持快捷键（后退/前进/上级/刷新/复制文件名/Ctrl+L/Ctrl+G 等）、鼠标手势和双击空白处跳上级——以“最近点击的面板”为操作目标，点哪个面板就操作哪个
- **优化：分屏中间分隔条更易拖动**：左右面板间的宽度调整手柄改为原生窗口并加宽，鼠标悬停时显示左右调整光标与可见抓取条，解决夹在内嵌资源管理器之间时光标不变、难以抓取的问题
- **优化：设置中可开关分屏快捷键**：`设置 → 手势与快捷键 → 快捷键设置` 新增 “F3 - 左右分屏对比” 条目，可单独启用/禁用

### v3.58 (2026-06-30)
- **主分支回归**：`spike/async-com` 验证内容已回归 `main`，后续开发默认在主分支进行
- **防卡顿：兜底轮询快照逐项 stat 封顶**：当前标签每 8 秒的目录变更兜底轮询，对超大本地目录会逐项 `stat` 造成周期性微卡顿；现超过上限即停止逐项统计，改用“总项目数 + 目录自身修改时间”兜底检测增删/重命名
- **防卡顿：大目录全选后右键不再逐项枚举**：右键菜单先用一次轻量 COM 计数短路，仅在恰好选中 1 项时才枚举，避免 `Ctrl+A` 选中海量文件后右键时对每项做 COM 查询造成的卡顿
- **性能：路径栏热路径去除无谓开销**：路径栏 `set_path` 在每次导航时构造的诊断字符串改为仅调试模式下生成，生产运行零开销
- **清理：移除旧状态栏遗留死代码**：状态栏早已重构为只显示 Git 摘要，相关无人调用的目录计数/选中项缓存/大小格式化等辅助代码及其缓存接线一并移除，纯瘦身不影响功能
- **新增：双击底部状态栏手动刷新重排**：路径栏偶发卡死不更新时，可双击底部状态栏/资源占用区域，触发当前标签刷新重新武装、路径栏强制重建，并模拟一次轻微 resize 级别的 UI 重排
- **状态栏资源占用颜色预警**：整机 CPU/内存占用 ≥75% 显示橙色、≥90% 显示红色，常态灰色，一眼看出系统是否吃紧
- **状态栏 CPU/内存改为整机占用**：状态栏右侧现示整机 CPU 占用与内存 已用/总量 GB（含百分比），便于判断系统是否还有剩余（GetSystemTimes + GlobalMemoryStatusEx，纯 ctypes）
- **稳定性：旧 QAx 路径读取看门狗硬超时**：仅回退到 Shell.Explorer 时生效，`LocationURL` 跨进程读取改在单 worker 线程（STA CoInitialize）上执行，250ms 硬超时即放弃并用缓存路径，UI 线程永不阻塞；IExplorerBrowser 仍走缓存值，不受影响

## 🛠️ 运行环境

**从可执行文件运行**（推荐）：
1. 从 [Releases](../../releases/latest) 下载 `TabExplorer.exe`
2. 双击运行即可
3. 程序会在 `TabExplorer.exe` 同目录读写以下持久化文件：
  - `config.json`：应用设置、固定标签、标题栏快捷方式、会话缓存
  - `bookmarks.json`：书签数据
  - `chat_history.json`：AI 聊天历史
  - `runtime_health.log`：资源快照日志（仅在设置中启用后生成）

**从源码运行**（需要 Python 3.9+）：
1. 双击 `0_install_requirements.bat` 安装依赖
2. 双击 `1_TabEx.bat` 启动程序
3. 若使用 AI 功能，确保 `requests` 已安装并在 `config.json` 或设置界面中填写 AI 服务配置

**开机自启动**：
将 `1_TabEx.bat` 的快捷方式放到 `shell:startup` 目录

## 📖 主要功能说明

### Everything 搜索（推荐）
1. 下载安装 [Everything](https://www.voidtools.com/)
2. 下载 [es.exe](https://www.voidtools.com/downloads/) 并放到 Everything 安装目录
3. 启动 Everything 并等待索引完成
4. 在 TabExplorer 中按 `Ctrl+F` 打开搜索，勾选"使用 Everything"即可享受毫秒级搜索

### 自动监听 Explorer 窗口
- 类似 Clover 的核心功能
- 新打开的 Windows 资源管理器窗口自动嵌入为标签页
- 可在设置中启用/禁用
- 若嵌入式 `Shell.Explorer` 控件初始化失败，会自动回退为使用系统资源管理器打开，避免生成异常标签页或拖慢界面

### 智能自动刷新
- 当前标签页会同时使用目录 watcher、文件 watcher 和低频轮询兜底检测文件变更
- 编辑当前目录里的文件后，列表、修改日期、大小等信息会更可靠地自动刷新
- 后台标签页不会持续高频轮询，只在切回时按需补刷新
- 这种策略兼顾了及时刷新和多标签场景下的性能开销
- 当当前目录正好是程序运行目录时，会自动忽略应用自身写出的配置/日志文件变化，避免 watcher 与会话快照互相触发导致刷新风暴

### 当前目录快捷定位
- 按 `Ctrl+G` 可输入关键字，快速检索当前目录下名称中包含关键字的文件夹和文件
- 只有一个匹配结果时会直接在当前目录中选中该项目
- 有多个匹配结果时会弹出表格选择窗口，展示名称、类型和完整路径
- 该快捷键可在设置界面中单独启用或禁用

### AI 助手
- 在 `设置 → AI 助手设置` 中配置启用开关、API 地址、API Key、模型名、面板宽度和系统提示词
- 标题栏 `🤖` 按钮仅在 AI 开关启用时显示
- 支持 OpenAI 兼容接口，也支持本地/内网模型服务
- AI 默认只在用户明确要求时输出操作指令，避免普通问答误操作

### 资源快照日志
- 在 `设置 → 调试设置` 中可启用“资源快照日志”
- 可设置采样间隔（分钟），默认用于低频长期观测，不建议设置过短
- 启用后程序会定时写入 `runtime_health.log`
- 可直接在设置中点击“打开资源日志”查看当前日志文件或日志目录
- 日志内容包括内存占用、线程数、标签页数量、搜索窗口数量、Toast 数量和 AI 工作线程状态

**当前支持的 AI 指令：**
- `OPEN_DIR`：打开或切换目录
- `READ_FILE`：读取文件内容，自动分段读取大文件
- `LIST_DIR`：列出目录内容
- `MKDIR`：创建目录
- `WRITE_FILE`：写入/覆盖文件
- `DELETE`：删除文件或目录（需确认）
- `RUN_SCRIPT`：运行脚本（需确认）

**安全限制：**
- AI 文件操作仅允许在当前标签页目录范围内执行
- 覆盖文件、删除文件/目录、运行脚本都需要用户手动确认
- 聊天面板支持拖拽本地文件进行快速阅读，单文件读取大小有限制

### 会话恢复
- 关闭软件时自动保存所有非固定标签页
- 下次启动时自动恢复，继续上次的工作
- 恢复顺序：先加载固定标签，再恢复缓存标签
- 可在设置中启用/禁用（默认开启）
- 打包版与源码版都会将会话缓存持久化到各自运行目录下的 `config.json`

### 书签管理
- 支持文件夹层级组织
- 导入/导出 JSON 格式书签
- 兼容 Chrome 书签格式
  

### 标签页历史
- 最多记录 20 个关闭的标签页
- `Ctrl+Shift+T` 快速恢复
- 或点击标签栏右侧的 ↶ 按钮

### 文件复制快捷键
- **Alt+Z** - 快速复制选中文件名（如果选择多个文件，会用逗号分隔）
- **Alt+X** - 快速复制文件路径\文件名（路径分隔符可在设置中配置为 `/` 或 `\`）
- 两个快捷键都支持多文件批量拷贝
- `Alt+Z`、`Alt+X`、`Ctrl+G` 等快捷键都可在设置界面中单独启用/禁用

```bash
# 1. 清空旧输出
Remove-Item build, dist -Recurse -Force -ErrorAction SilentlyContinue

# 2. 运行打包脚本
.\2_build_exe.bat
```

### 项目文件说明
- `TabEx.py` - 主程序入口
- `requirements.txt` - Python 运行依赖列表
- `0_install_requirements.bat` - 依赖安装脚本
- `1_TabEx.bat` - 源码运行脚本
- `2_build_exe.bat` - PyInstaller 打包脚本
- `generate_icon.py` - 图标生成脚本（需要 Pillow：`pip install pillow`）

> `config.json`、`bookmarks.json`、`chat_history.json`、`runtime_health.log` 均为运行时自动生成的本地数据文件，已在 `.gitignore` 中忽略，不随仓库分发。
